## 前置准备

### 1. 配置环境变量

在项目根目录的 `.env.local` 文件中添加以下配置：

```env
# 令牌签名密钥（必须至少 32 个字符）
TOKEN_SECRET=your-very-long-secret-key-at-least-32-characters

# 管理员 API 密钥（用于调用令牌生成接口）
ADMIN_API_KEY=your-admin-api-key
```

> ⚠️ **重要提示：**
> - `TOKEN_SECRET` 是令牌签名的核心密钥，泄露后任何人都能伪造令牌
> - `ADMIN_API_KEY` 是管理员接口的访问凭证
> - 两个密钥都应该使用随机字符串，不要使用简单密码

### 2. 启动服务

```bash
npm run dev
# 或生产环境
npm run build && npm start
```

## 令牌参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `plan` | string | `pro` | 套餐类型：`pro` 或 `enterprise` |
| `tier` | string | `standard` | 子套餐等级，自定义标识 |
| `subject` | string | 随机生成 | 用户唯一标识，用于追踪令牌归属 |
| `validDays` | number | `365` | 有效期（天） |
| `generateQuota` | number | `200` | 每日生成 Issue 的次数上限 |
| `expandQuota` | number | `-1` | 每日智能扩充次数上限，`-1` 表示使用默认值 |
| `meta` | object | `{}` | 附加元数据，自定义字段 |

#### 使用 curl

```bash
curl -X POST http://localhost:3000/api/admin/generate-token \
  -H "Content-Type: application/json" \
  -H "x-admin-key: your-admin-api-key" \

    "subject": "user_001",
    "validDays": 365,
```

```powershell
$headers = @{
  "Content-Type" = "application/json"
  "x-admin-key" = "your-admin-api-key"

$body = @{
  plan = "pro"
  tier = "standard"
  subject = "user_001"
  validDays = 365
  generateQuota = 200
  expandQuota = 50

Invoke-RestMethod -Uri "http://localhost:3000/api/admin/generate-token" `
  -Method POST -Headers $headers -Body $body
```

### 方式二：通过 Node.js 脚本生成

在项目根目录创建 `scripts/generate-token.js`：

```javascript
// scripts/generate-token.js
const crypto = require('crypto');

const TOKEN_PREFIX = 'ie_';
const TOKEN_VERSION = 'v1';

function getSecret() {
  const secret = process.env.TOKEN_SECRET;
  if (!secret || secret.length < 32) {
    throw new Error('TOKEN_SECRET environment variable is required (min 32 chars)');
  }
  return secret;
}

function base64urlEncode(buf) {
  return buf.toString('base64url');
}

function sign(payloadB64) {
  const secret = getSecret();
  return crypto.createHmac('sha256', secret).update(payloadB64).digest('base64url');
}

function generateToken(opts = {}) {
  const now = Math.floor(Date.now() / 1000);
  const validDays = opts.validDays || 365;
  
  const payload = {
    sub: opts.subject || crypto.randomBytes(8).toString('hex'),
    plan: opts.plan || 'pro',
    tier: opts.tier || 'standard',
    iat: now,
    exp: now + validDays * 86400,
    quotas: {
      generate: opts.generateQuota ?? 200,
      expand: opts.expandQuota ?? -1
    },
    meta: opts.meta
  };

  const payloadStr = JSON.stringify(payload);
  const payloadB64 = base64urlEncode(Buffer.from(payloadStr));
  const signature = sign(payloadB64);

  return `${TOKEN_PREFIX}${TOKEN_VERSION}_${payloadB64}.${signature}`;
}

// 读取命令行参数
const args = process.argv.slice(2);
const options = {};

args.forEach(arg => {
  const [key, value] = arg.split('=');
  if (key && value) {
    const cleanKey = key.replace(/^--/, '');
    if (['validDays', 'generateQuota', 'expandQuota'].includes(cleanKey)) {
      options[cleanKey] = parseInt(value, 10);
    } else {
      options[cleanKey] = value;
    }
  }
});

const token = generateToken(options);
console.log('\n✅ 令牌生成成功！\n');
console.log(`令牌: ${token}\n`);
console.log(`长度: ${token.length} 字符\n`);
```

使用方法：

```bash
# 设置环境变量（Windows PowerShell）
$env:TOKEN_SECRET="your-very-long-secret-key-at-least-32-characters"

# 生成默认 Pro 令牌（365天，200次生成/天）
node scripts/generate-token.js

# 自定义参数
node scripts/generate-token.js \
  --plan=pro \
  --subject=user_001 \
  --validDays=365 \
  --generateQuota=200 \
  --expandQuota=50
```

### 方式三：通过 TypeScript 代码生成

在你的项目中直接使用 `lib/token.ts`：

```typescript
import { generateToken, verifyToken } from './lib/token'

// 生成 Pro 版令牌
const proToken = generateToken({
  plan: 'pro',
  subject: 'user_001',
  validDays: 365,
  generateQuota: 200,
  expandQuota: 50
})

console.log('Pro Token:', proToken)

// 生成企业版令牌（更高配额）
const enterpriseToken = generateToken({
  plan: 'enterprise',
  subject: 'company_001',
  validDays: 730,
  generateQuota: 1000,
  expandQuota: -1,  // -1 表示使用默认上限
  meta: {
    company: 'Acme Corp',
    contact: 'admin@example.com'
  }
})

console.log('Enterprise Token:', enterpriseToken)

// 验证令牌
const result = verifyToken(proToken)
if (result.valid) {
  console.log('令牌有效:', result.payload)
} else {
  console.log('令牌无效:', result.error)
}
```

## 常见问题

### Q1: 令牌可以修改吗？

**不可以。** 令牌使用 HMAC-SHA256 签名，任何修改都会导致签名验证失败。

### Q2: 如何吊销令牌？

当前版本使用内存计数器，服务重启后计数会重置。如果需要真正的令牌吊销机制，建议：

- 将令牌存储在数据库中，增加 `revoked` 字段
- 每次请求时检查令牌是否被吊销
- 令牌过期时间设短一些（如 30 天）

### Q3: 不同用户可以生成相同的令牌吗？

理论上不会，因为每个令牌包含随机的 `sub`（用户标识）和不同的 `iat`（签发时间），所以每个令牌都是唯一的。

### Q4: 令牌格式是什么样的？

```
ie_v1_eyJzdWIiOiJ1c2VyXzAwMSIs...AbCdEf123456
└──┘ └─┘ └──────────────────────────┘ └────────┘
  │   │              │                      │
  │   │              │                      └─ HMAC-SHA256 签名 (base64url)
  │   │              └─ 载荷数据 (base64url 编码的 JSON)
  │   └─ 令牌版本号 (v1)
  └─ 令牌前缀 (ie_)
```

### Q5: 令牌安全吗？

- ✅ 使用 HMAC-SHA256 签名，无法伪造
- ✅ 签名验证使用 `crypto.timingSafeEqual` 防止时序攻击
- ⚠️ 载荷是 base64 编码的，可以被解码查看（但无法修改）
- ⚠️ 不要在令牌中存储敏感信息（如密码、密钥等）

### Q6: 如何生成安全的密钥？

```bash
# 生成 32 字节随机密钥（推荐）
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 或者使用 openssl
openssl rand -hex 32
```

### Q7: 免费用户的扩充次数如何调整？

修改 `app/api/expand/route.ts` 中的常量：

```typescript
const FREE_DAILY_EXPAND_LIMIT = 1  // 改为你想要的次数
```

## 技术实现细节

令牌系统的核心代码位于 [lib/token.ts](file:///g:/桌面/github/lib/token.ts)：

- `generateToken()` - 生成令牌
- `verifyToken()` - 验证令牌签名和有效期
- `isValidPaidToken()` - 检查是否为有效付费令牌（向后兼容旧格式）
- `getTokenQuota()` - 获取令牌配额
- `getTokenPlan()` - 获取令牌套餐类型
